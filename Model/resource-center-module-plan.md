# 方案资料、能力说明与项目案例沉淀模块规划

## 目标定位

这个模块不是资料列表，而是首页上的内容中枢，用来回答三个问题：

1. 平台能做什么
2. 平台做过什么
3. 用户下一步怎么推进

因此它应同时承担平台能力展示、案例背书和售前转化入口三个职责。

## 内容结构

### 1. 方案资料

面向“我要了解平台整体能力”的用户。

建议放置内容：

- 平台能力总览
- 行业解决方案手册
- 算法能力清单
- 部署与交付流程说明
- 方案白皮书

展示重点：

- 强调平台视角
- 强调适用行业
- 强调可下载、可查看、可沟通

### 2. 能力说明

面向“我要评估技术可行性”的用户。

建议放置内容：

- 算法分类说明
- 适用场景与输入输出说明
- 在线检测操作指南
- 结果导出说明
- API / SDK 接入说明

展示重点：

- 强调结构化说明
- 强调操作门槛低
- 强调从体验到接入的链路

### 3. 项目案例

面向“我要判断是否可落地”的用户。

建议放置内容：

- 输电巡检案例
- 配网识别案例
- 光伏缺陷检测案例
- 安全监管案例
- 环保治理案例

每个案例建议至少包含：

- 行业场景
- 解决的问题
- 上线效果或收益

### 4. 售前与合作支持

面向“我要推进下一步”的用户。

建议放置内容：

- 资料申请
- 测试样例提交规范
- 演示预约
- 合作咨询入口
- 商务联系方式

## 首页展示建议

首页不适合把所有资料平铺展开，更适合做成以下结构：

1. 左侧重点内容卡
   - 展示“平台资料总览”或“重点解决方案”
   - 作用是快速建立平台认知

2. 右侧三类内容分区
   - 方案资料
   - 能力说明
   - 项目案例

3. 底部售前支持入口
   - 资料申请
   - 提交测试样例
   - 商务咨询

## 每条内容建议字段

- 标题
- 类型标签
- 一句话说明
- 适用场景或更新时间
- 操作入口

## 首页文案方向

模块标题建议保留：

- 方案资料、能力说明与项目案例沉淀

辅助说明建议突出：

- 内容服务于售前沟通、能力评估和项目推进
- 不是单纯资料堆叠，而是分阶段支持用户决策
