# 视觉算法平台首页改造方案

## 1. 改造目标

当前首页的问题在于信息组织偏“汇报页”：

- 模块平均铺陈，缺少绝对主视觉
- 内容重点在“平台有什么”，而不是“客户为什么要继续看”
- 视觉切分依赖卡片和容器，页面节奏碎
- 首页更像内部展示页，不像面向客户的商业官网首页

本次改造目标：

- 首页整体采用商业化官网表达，而不是系统后台表达
- 页面首屏铺满屏幕，建立品牌感和行业感
- 各模块之间不依赖大块卡片容器切割，而是通过背景、留白、排版和内容密度自然过渡
- 首页叙事围绕“价值建立 -> 能力证明 -> 产品展示 -> 行业落地 -> 商务转化”

---

## 2. 首页整体设计方向

### 2.1 关键词

- 商业化
- 品牌感
- 大场景
- 无缝分区
- 深色首屏
- 结果导向

### 2.2 页面气质

建议采用“深色品牌首屏 + 浅色内容区 + 深色页脚”的结构：

- 首屏使用深红棕到暗黑渐变背景，增强品牌张力和高级感
- 中段能力与产品内容切换为浅底，提升阅读效率
- 行业方案区可加入轻微暖色/品牌色渐变，作为中段情绪转场
- 页脚回归深色，形成页面收束

### 2.3 布局原则

- 页面整体采用 full-width 铺满屏幕，不做“大白底中间一张卡”的后台式布局
- 模块与模块之间尽量无边框衔接
- 每个区块都以“背景变化 + 栅格变化 + 标题层级变化”来完成区分
- 控制内容块的最大宽度，但背景必须铺满浏览器宽度
- 减少玻璃卡片、悬浮卡片、描边容器的大面积使用

以下结构按用户从上到下浏览顺序设计。

### 结构

- 左侧：企业 Logo + 平台名称
- 中间：导航菜单
- 右侧：主 CTA 按钮

### 导航建议

- 首页
- 算法能力
- 行业方案
- 在线体验
- 联系我们

### 按钮建议

- 主按钮：`预约演示`
- 次级文字入口：`进入算法商城`

### 设计说明

- 导航悬浮在首屏上方，采用半透明或轻量描边
- 高度收紧，不要做成后台导航条
- 用户中心、登录状态这类平台功能入口不要在首页做主角色，可弱化到右上角次级位置或后续页面

- 你是谁
- 你做什么
- 你服务哪些场景
- 下一步可以做什么

```text
│ 主按钮 + 次按钮                              │
│                                              │
│ 下方横向数据指标带                           │
└──────────────────────────────────────────────┘
```

### 文案方向

#### 主标题方向 A

`智能视觉，驱动电力场景商业落地`

#### 主标题方向 B

`面向电力巡检与行业治理的视觉算法平台`

#### 主标题方向 C

`让视觉算法从能力展示走向项目交付`

#### 副标题方向

`聚焦输电、配网、光伏、安全监管与应急场景，提供算法展示、在线验证、方案对接与项目落地的一体化平台。`

### Hero 搜索/入口建议

输入框 placeholder 建议：

- `搜索算法、场景或解决方案`

快捷标签建议：

- 输电巡检
- 配网巡检
- 光伏巡检
- 安全监管
- 应急救援

### CTA 建议

- 主按钮：`预约方案演示`
- 次按钮：`查看精选算法`

### 首屏指标建议

建议以一行四列展示，不使用独立大卡片，仅通过分隔线和数字层级区分：

- `160+` 算法能力资产
- `100+` 场景样例数据
- `6000+` 测试任务沉淀
- `90%+` 重点模型识别精度

### 设计说明

- 首屏高度建议至少 `100vh`
- 左右分栏，左文案右视觉
- 右侧可以使用算法识别框叠加行业图片，而不是纯轮播图
- 当前首页的轮播逻辑可以保留，但表现形式需要更像“品牌主视觉”

## 3.3 核心能力区

### 目标

向客户证明这不是单点算法展示，而是完整能力体系。

### 线框结构

```text
┌──────────────────────────────────────────────┐
│ 小标题 + 区块标题 + 说明                      │
│                                              │
│ 三列能力项                                   │
│ 01 算法资产沉淀                              │
│ 02 在线测试验证                              │
│ 03 行业方案交付                              │
└──────────────────────────────────────────────┘
```

### 文案方向

#### 区块标题

`核心技术能力`

#### 区块说明

`围绕算法资产管理、在线验证和行业化交付，构建可展示、可验证、可承接的平台能力闭环。`

#### 三项能力文案

1. `算法资产沉淀`
   `按检测、分割、多模态和异常分析等方向组织算法能力，支持面向客户的标准化展示。`

2. `在线测试验证`
   `支持样本上传、参数配置、结果预览与报告输出，缩短售前沟通与效果验证路径。`

3. `行业方案交付`
   `结合电力与行业场景需求，将单项算法能力快速组合为可沟通、可落地的解决方案。`

### 设计说明

- 不建议再用厚重卡片包裹
- 每个能力项可采用“图标 + 标题 + 一小段说明”的轻结构
- 通过浅色背景、分栏、数字编号完成区分

## 4. 页面无缝拼接设计原则

你特别强调“不要用卡片框架切割，而要无缝拼接”，这一点建议按以下原则执行。

## 4.1 用背景区分，而不是用容器框住

建议的背景节奏：

- Hero：深色渐变背景
- 核心能力：浅灰白背景
- 精选算法：纯白背景
- 行业方案：浅暖色渐变背景
- 商务转化 + 页脚：深色背景

这样区块天然分层，不需要大边框和大卡片。

## 4.2 用留白区分，而不是用描边区分

- 每个区块上下留白统一拉开
- 标题与内容之间保持明确层级
- 模块之间通过垂直节奏过渡，不靠边框硬切

## 4.3 用排版和密度变化制造节奏

- Hero 大标题、大留白、高对比
- 能力区三列均匀排布，强调秩序
- 算法区图像密度更高，增强产品感
- 行业区回归轻量入口，形成节奏放松
- 商务区再集中收口

## 4.4 卡片只作为“内容单元”，不是“区块容器”

允许存在：

- 算法项
- 行业入口项
- 联系方式项

不建议存在：

- 整个区块再套一层大白卡
- 连续多个区块都靠大圆角白卡包裹

---

## 5. 文案总原则

### 5.1 文案语气

建议从“汇报式说明”改为“商业化表达”：

- 少用：平台围绕、平台支持、平台提供
- 多用：聚焦、面向、覆盖、支撑、推动、落地

### 5.2 文案长度

- 主标题控制在 12 到 18 个字
- 副标题控制在 1 句
- 区块说明控制在 1 句
- 卡片说明控制在 1 到 2 句

### 5.3 文案重心

首页优先讲：

- 场景
- 能力
- 指标
- 价值
- 行动

首页弱化讲：

- 平台内部模块说明
- 操作流程细节
- 资讯列表
- 用户中心逻辑

---

## 6. 对当前首页的具体调整建议

基于当前 [index.html](./index.html) 结构，建议这样调整：

### 保留

- 首页 Hero 的动态展示能力
- 热门算法推荐的数据结构
- 行业方案的数据结构
- 商务联系方式内容

### 弱化或移出首页

- 资讯与资源中心降级
- 用户中心不作为首页重点入口
- 平台能力总览改写为商业化“核心技术能力”

### 重构

- 首屏布局重构为真正满屏 Hero
- 统计数字整合进 Hero 下方
- 所有区块取消“大卡片包裹式”的外层容器感
- 背景、分区、图文比例全部重新整理
